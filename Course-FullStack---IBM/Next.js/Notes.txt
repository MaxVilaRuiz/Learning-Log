Basic Commands:

To initialize a Next.js project:
> npx create-next-app@latest <project_name>

To run the website:
> npm run dev

Project Structure:
.
├── app
│   ├── favicon.ico
│   ├── globals.css
│   ├── layout.tsx
│   └── page.tsx
├── eslint.config.mjs
├── next-env.d.ts
├── next.config.ts
├── package-lock.json
├── package.json
├── postcss.config.mjs
├── public
│   ├── file.svg
│   ├── globe.svg
│   ├── next.svg
│   ├── vercel.svg
│   └── window.svg
├── README.md
├── tsconfig.json
└── types
    ├── cache-life.d.ts
    ├── routes.d.ts
    └── validator.ts

package.json: manifest of the project. Includes dependencies, scripts...
next.config.mjs: advance configs of Next.js
tsconfig.json: TypeScript config, which includes import routes, strict typo rules...
tailwind.config.mjs: TailwindCSS config, which includes fonts, breakpoints, color palettes...
public/: static assets folder, which must include static images, icons, favicons...
app/: the core of the project.
app/layout.tsx: root of the website, which must include elements that would repeat in every page, such as: global typo, shared components, basic html labels...
app/page.tsx: the '/' router (landing page).
app/globals.css: global styles.
-> Each page in app/ creates a website route.
